## 1. Design Philosophy

### Core Principles

CycleAlarm follows the principle of **"boring but trustworthy"** â€” users interact with this app infrequently to set recurring tasks, then forget it exists until the alarm saves them from missing something important. The UI must feel reliable, not flashy.

| Principle               | Description                          | Implementation                          |
| ----------------------- | ------------------------------------ | --------------------------------------- |
| **Clarity Over Flash**  | Time-to-value under 2 seconds        | Single-screen home, no hamburger menus  |
| **Deliberate Actions**  | Prevent accidental alarm dismissal   | Slide-to-stop, hold-to-dismiss patterns |
| **Context Awareness**   | Adapt to time of day and user state  | Cool tones at night, warm in morning    |
| **Platform Respect**    | iOS feels iOS, Android feels Android | Liquid Glass vs Material 3, not hybrid  |
| **Accessibility First** | WCAG 2.2 AA compliance               | 44pt/48dp touch targets, 4.5:1 contrast |
