### Bug Fixes

- **Alarm scheduling after completion** — Removed past-due filter in `rescheduleAllAlarms()` that was skipping alarms whose `nextFireDate` had already passed, preventing them from being rescheduled after completion
- **Silent scheduling failures** — Replaced `try?` with `do/catch` + error logging in alarm scheduling paths so failures are no longer silently swallowed
- **Persistent nextFireDate updates** — Added `saveChanges()` call to `AlarmRepository` to ensure `nextFireDate` updates are persisted to the store

### Feature Gating

- **Notes tier gating** — Notes field now shows a lock icon for free-tier users with a tap-to-upgrade paywall trigger. Plus users retain full access.
